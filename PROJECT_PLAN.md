# 艺术家个人资产体系 · 项目规划

> 版本：v1.0 · 2026-06-26  
> 阶段：**验证期 MVP** — 聚焦售前，不含链上/NFT

---

## 1. 项目定位

### 一句话

**艺术家售前阶段的价值发现与资产沉淀系统** — 不是先卖作品，而是先让艺术家的价值被记录、解释、跟踪与支持。

### 核心命题

> 一个艺术家在进入市场、被正式定价之前，如何被更透明、更持续、更面向大众地**理解、评估、记录和积累价值**。

### 我们不是什么

- 不是卖画商城 / 艺术品电商
- 不是艺术金融 / 估值投资平台
- 不是 NFT / 区块链平台（**后期可选扩展**）
- 不是「艺术家包装公司」

### 我们是什么

| 对外说法 | 含义 |
|----------|------|
| 艺术家个人资产基础设施 | 作品、履历、过程、关系结构化沉淀 |
| 艺术家售前价值管理平台 | 成交前的价值形成，而非成交本身 |
| 艺术家长期价值管理工具 | 可验证、可积累、可转化 |

---

## 2. 核心痛点

表面是「大众不懂艺术」，本质是：

> **艺术家价值形成机制过于封闭，定价权、解释权和流通权被少数中介垄断。**

| 层级 | 问题 |
|------|------|
| 解释权 | 价值叙事依赖画廊、策展人、学院、拍卖行，大众难以参与理解 |
| 定价权 | 价格靠关系、履历、背书、圈层稀缺，不透明 |
| 资产化缺失 | 早期创作信号（持续性、风格演化、社群认可等）未被系统记录 |
| 软资产未定价 | 风格、叙事、声誉、社群、过程材料等很少被结构化展示 |

---

## 3. 硬资产与软资产

### 硬资产

可持有、可管理、可（后期）交易的部分：

- 原作、限量版、雕塑/装置
- 数字作品、草图原件、手稿
- 展览衍生品、库存作品
- 版次管理、已售/未售状态

### 软资产（差异化壁垒）

| 类型 | 内容 |
|------|------|
| 创作资产 | 风格一致性、系列完整度、创作频率、概念深度 |
| 叙事资产 | 自述、系列世界观、评论/策展文本 |
| 声誉资产 | 展览/驻留/合作履历、机构背书 |
| 社群资产 | 关注者质量、支持者留存、参与度 |
| 过程资产 | 草图、创作日志、研究资料、幕后影像 |
| 市场资产 | 成交记录、版次控制（后期） |

目标：把零散信息变成 **可积累、可展示、可验证、可转化** 的资产。

---

## 4. 产品四层架构

| 层级 | 内容 | 验证期 |
|------|------|--------|
| **L1 身份与档案** | 主页、认证、履历、作品/系列归档、时间线 | ✅ MVP 核心 |
| **L2 叙事与解释** | 系列关联、风格演化、关键节点、大众导览 | ✅ 差异化核心 |
| **L3 支持者关系** | 关注、早期支持者、长期跟踪、参与权与见证权 | ✅ 轻量启动 |
| **L4 定制化产品化预售** | 按系列/项目设计的预售 SKU，非标准化发币 | 🔜 验证后期 |
| **L5 交易与转化** | 原作、版画、授权、联名等完整成交 | ⏸ 更后期 |

**原则**：价格是系统的结果之一，不是起点。先做价值发现与价值记录，不做「大众定价模型」。

## 5. 分阶段路线图

### 第一阶段：艺术家个人资产档案（当前）

**目标**：回答「一个艺术家在被市场正式定价之前，如何被记录？」

- 作品档案（硬资产底座）
- 创作脉络与系列结构
- 过程文档（草图、笔记、工作室影像）
- 支持者关系（关注、早期名单）
- 公开解释层（面向大众的非学院化叙事）
- 可验证记录（时间戳、版本、发布记录）

**交付形态**：每位艺术家一个「资产档案页」（6 模块，见 [docs/product/mvp-modules.md](./docs/product/mvp-modules.md)）

**收费逻辑（验证期）**：档案建档服务 / 高级会员，不卖估值、不卖投资。

---

### 第二阶段：成长与资产可视化

- 资产图谱、时间轴、风格变化
- 社群信号、作品关联
- 形成平台自己的「评估语言」（非单一打分，而是资产形成度指标）

---

### 第三阶段：定制化 · 产品化预售

> **替代原「标准化发行/NFT」思路**：不做见作品就铸凭证，而是为每位艺术家/每个系列**定制可售卖的预售产品包**。

#### 什么是「产品化预售」

把售前关系转化为 **有明确边界、有语义、有交付物** 的预售 SKU，例如：

| 预售产品类型 | 说明 | 示例 |
|--------------|------|------|
| 系列文献包 | 系列介绍 + 草图 + 创作手记 + 限量编号 | 「静安里系列 · 创作文献包 #12/50」 |
| 创作计划支持包 | 支持某阶段创作，含更新权益与见证记录 | 「2026 春季创作计划 · 支持者档」 |
| 工作室体验包 | 开放日、导览、幕后访问资格 | 「工作室开放日 · 单次预约」 |
| 实体优先权包 | 未来原作/版画的优先认购权（非证券化） | 「系列 A · 优先认购登记」 |
| 联名/定制意向包 | 品牌合作、空间驻留等意向金 | 「联名方案咨询 · 意向登记」 |

#### 定制化原则

- **一人一策 / 一系列一策**：由平台协助艺术家定义 SKU，而非统一模板发「token」
- **语义清晰**：每个预售产品写清交付物、权益边界、不退款规则（如适用）
- **与档案上下文绑定**：预售页必须链接到 L1/L2 资产档案，有叙事支撑
- **轻量起步**：验证期仅 1–2 种 SKU 模板，人工配置为主

#### 收费

- 预售产品设计与上架服务费
- 成交/意向金抽成（后期接支付）

#### 明确不做（本阶段）

- 标准化 NFT 铸造
- 升值/炒作叙事
- 无上下文的「发币式」发行

---

### 第四阶段：完整交易与转化（远期）

原作、版画、授权、联名、分账等 — 在前三层与产品化预售验证通过后再展开。

## 6. 目标用户

### 优先服务（C 端）

**新锐但认真创作的独立艺术家**

- 有稳定创作与一定作品量
- 缺少系统呈现，尚未被强势画廊绑定
- 作品散落在各平台，无完整时间线
- 愿意整理过程资产、建立长期档案

### 次级用户

- 跨界创作者（插画、新媒体、实验影像等）
- 小型画廊 / 独立空间 / 策展项目（B 端工具，后期）

### 不优先

- 已被大画廊绑定的成熟艺术家
- 纯业余、无持续产出者

## 8. 商业模式（验证期 → 成长期）

| 阶段 | 收入来源 |
|------|----------|
| 验证期 | 免费/低价共建；档案整理服务费 |
| 成长期 | 艺术家会员订阅；高级叙事服务；预售产品配置费 |
| 成熟期 | 预售/交易抽成；机构版 SaaS |

**验证期明确不卖**：估值、投资艺术家、艺术金融。

## 10. 落地策略

1. **先做产品化服务，再大平台**：人工服务 5–20 位艺术家，抽象产品
2. **第一单流程**：深度访谈 → 资产地图 → 档案页原型 → 第三方理解度验证
3. **四个验证问题**：是否愿整理过程资产？是否愿付费维护？第三方是否更易理解？是否自然长出支持关系？
4. **预售验证**：选 1 位艺术家做 1 个「系列文献包」或「创作计划支持包」试点

## 12. 与旧项目（数字创作者经济平台）的关系

旧项目探索了「AI 创作者 + 工厂 + 评级 + 合作」方向。  
**本项目聚焦**：艺术家个人资产化 + 售前价值发现；工厂/AI 创作变现等痛点可后续作为辅助场景接入，不作为验证期核心。

代码与开发环境已迁移至本目录，UI 与数据模型将按新方向逐步调整。
