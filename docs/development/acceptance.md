# 验收标准

> 验证期 MVP

---

## 1. 产品验收

| # | 标准 | 验证方式 |
|---|------|----------|
| A1 | 艺术家档案页包含 6 模块核心字段 | 页面走查 |
| A2 | 系列与时间线可展示创作脉络 | 用户测试 |
| A3 | 过程资产可上传并分级展示 | 功能测试 |
| A4 | 第三方看完后能说出创作线索与代表系列 | 5 人理解度测试 |
| A5 | 移动端可用 | 真机/窄屏 |

---

## 2. 技术验收

| # | 标准 |
|---|------|
| T1 | `pnpm build` 通过 |
| T2 | MySQL 连接与基础 CRUD |
| T3 | 本地图片上传与访问 |
| T4 | 管理端可录入评级 |

---

## 3. 业务验证（人工）

| # | 标准 |
|---|------|
| B1 | ≥3 位艺术家愿意共建档案 |
| B2 | ≥1 位愿意为档案维护付费或深度投入时间 |
| B3 | 至少策划 1 个「系列文献包」预售页（可不成交） |

---

## 4. 明确不验收

- NFT/链上功能
- 完整支付闭环
- 算法化艺术评分
