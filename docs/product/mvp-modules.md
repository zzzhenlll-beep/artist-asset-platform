# 资产档案页 · 6 模块（MVP）

> 验证期最小交付：每位艺术家一个「资产档案页」

---

## 模块 1：基础身份

| 字段 | 说明 |
|------|------|
| 姓名 / 艺术名 | 展示名 |
| 城市 | 可选 |
| 主要媒介 | 绘画、装置、影像等 |
| 创作关键词 | 3–5 个 |
| 个人陈述 | 艺术家自述 |
| 社媒 / 联系 | 外链 |

---

## 模块 2：作品档案（硬资产底座）

结构化字段，非简单相册：

| 字段 | 说明 |
|------|------|
| 作品名 | |
| 年份 | |
| 尺寸 / 媒介 | |
| 系列归属 | 关联系列 ID |
| 创作说明 | |
| 状态 | 已售 / 未售 / 不公开 |
| 版次 | 如有 |
| 是否开放支持/预售咨询 | 链到 L4 产品化预售（后期） |

---

## 模块 3：系列与时间线

与普通作品集的核心差异：

- **年度时间轴**：按时间展示关键节点
- **系列关系**：系列之间的主题关联
- **过程链**：草图 → 成品（可选）
- **主题演化路径**：风格/议题如何发展

---

| 类型 | 示例 |
|------|------|
| 草图 / 手稿 | |
| 创作笔记 | |
| 参考资料 | |
| 工作室影像 | |
| 实验稿 | |
| 展览筹备文档 | |

**可见性分级**：

- 公开
- 支持者可见（后期）
- 仅艺术家私有存档

---

## 模块 5：支持者关系

验证期轻量实现：

- 关注 / 订阅
- 早期支持者名单（可手动维护）
- 某系列支持者（后期）
- 联系 / 合作咨询入口

> 先建立长期跟踪关系，不急于售卖。

---

## 模块 6：可验证记录

与「普通作品集网站」的差异层：

| 记录类型 | 说明 |
|----------|------|
| 首次公开时间 | 作品/系列上线时间 |
| 版本更新 | 档案修订记录 |
| 系列创建时间 | |
| 文本发布时间 | 策展/自述等 |
| 支持 / 咨询记录 | 后期 |

验证期用数据库时间戳 + 操作日志即可，无需链上。

---

## 页面信息架构（建议路由）

| 路由 | 内容 |
|------|------|
| `/` | 平台首页 / 精选艺术家 |
| `/artists/[id]` | 艺术家资产档案页（6 模块聚合） |
| `/artists/[id]/works/[workId]` | 单件作品详情 |
| `/artists/[id]/series/[seriesId]` | 系列与时间线 |
| `/artists/[id]/timeline` | 创作时间轴 |
| `/dashboard` | 艺术家后台（维护档案） |
| `/admin` | 运营：评级、预售产品配置（后期） |

> 当前代码仍沿用旧路由（`/creators`、`/works`），后续迭代时对齐上表。
