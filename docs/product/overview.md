# 产品愿景

> 艺术家售前阶段的价值发现系统

---

## 1. 我们要解决什么

不是让大众「给艺术打分」，而是：

> 让艺术家的价值在被正式定价之前，就能被**更多人看见、理解、记录和持续跟踪**。

当前市场的问题：

- **解释权**集中在画廊、策展人、学院
- **定价权**不透明，依赖关系与背书
- **早期信号**（创作持续性、风格演化、社群认可）没有资产化
- **软资产**（叙事、声誉、过程）无法被结构化展示

---

## 2. 售前阶段的三件事

### ① 建立艺术家资产账户

每个艺术家是一个「资产体」，不是简单作品集：

- **硬资产**：作品库存、版次、媒介/尺寸/状态
- **软资产**：风格路径、主题谱系、履历、文本、社群与支持者数据

### ② 从「卖作品对象」到「可跟踪的文化资产体**

展示：

- 正在形成怎样的资产结构
- 创作方向如何演进
- 谁在早期支持、哪些是关键节点
- 哪些软资产正在积累

### ③ 让大众参与「理解」

设计公众解释层 — 不是简单点赞，而是：

- 理解创作线索与系列关系
- 选择长期跟踪的方向
- 选择支持的形态（关注、文献包、工作室访问等）

---

## 3. 产品四层（验证期聚焦 L1–L3）

```
L1 身份与档案  ──►  MVP 核心
L2 叙事与解释  ──►  差异化
L3 支持者关系  ──►  轻量启动
L4 定制化产品化预售  ──►  验证后期（见 presale-playbook.md）
L5 交易与转化  ──►  远期
```

---

## 4. 关键原则

| 原则 | 说明 |
|------|------|
| 价值发现先于定价 | 不做「大众估值模型」 |
| 可验证 | 时间线、版本、记录，不是空讲故事 |
| 可积累 | 每次创作、展览、文本都沉淀进档案 |
| 可转化 | 沉淀可导向支持、预售、合作（后期） |
| 服务先于平台 | 验证期人工建档 + 半工具化 |

---

## 5. 目标用户

**新锐独立艺术家**：有持续创作、缺系统呈现、未被大画廊绑定。

典型痛点：作品散落各平台、无时间线、文本与作品脱节、被看见靠运气。

---

## 6. 对外表述

**推荐**：

> 我们在做艺术家售前阶段的价值发现系统 — 先记录、解释、跟踪与支持，再谈交易。

**避免**：

艺术估值平台、艺术金融、NFT 平台、去中心化画廊。
