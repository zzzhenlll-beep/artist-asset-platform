# 定制化 · 产品化预售手册

> 第三阶段能力 · 验证后期试点  
> **替代思路**：不做标准化 NFT/链上发行，而为每位艺术家定制可售卖的预售 SKU。

---

## 1. 为什么叫「产品化预售」

| 旧思路（不做） | 新思路（做） |
|----------------|--------------|
| 见作品就铸凭证 | 按系列/项目设计预售产品包 |
| 统一 token 模板 | 一人一策、一系列一策 |
| 升值/稀缺炒作叙事 | 语义清晰的交付物与权益 |
| 与档案脱节 | 必须绑定 L1/L2 资产上下文 |

**产品化** = 每个预售项是明确的 SKU：有名称、交付物、边界、价格带、库存/名额。  
**定制化** = 平台协助艺术家从档案中提炼可售卖的支持形态，非统一发「会员币」。

---

## 2. 预售 SKU 类型（模板库）

验证后期从 **1–2 种** 起步，人工配置为主。

### A. 系列文献包

| 项 | 内容 |
|----|------|
| 交付物 | 系列介绍 PDF/页、精选草图、创作手记、工作室影像、限量编号 |
| 定价 | 低档（如 99–499 元） |
| 名额 | 限量编号，如 50 份 |
| 适用 | 有完整系列、过程材料丰富的艺术家 |

### B. 创作计划支持包

| 项 | 内容 |
|----|------|
| 交付物 | 阶段创作更新、幕后记录、支持者名单公示、完成后的文献包权益 |
| 定价 | 中档（如 500–3000 元） |
| 周期 | 绑定某一创作计划（如 3–6 个月） |
| 适用 | 有明确下一阶段创作计划的艺术家 |

### C. 工作室体验包

| 项 | 内容 |
|----|------|
| 交付物 | 开放日预约、导览、小型座谈 |
| 定价 | 按场次 |
| 名额 | 严格限人数 |
| 适用 | 有实体工作室、愿意做面对面连接的艺术家 |

### D. 实体优先权包

| 项 | 内容 |
|----|------|
| 交付物 | 未来某系列原作/版画的**优先认购登记权**（非证券化、非承诺升值） |
| 定价 | 登记费或支持档 |
| 注意 | 法律文案需明确「优先权≠购买义务」 |

### E. 联名 / 定制意向包

| 项 | 内容 |
|----|------|
| 交付物 | 品牌联名、空间驻留、定制项目的意向登记 + 平台协助对接 |
| 定价 | 意向金或咨询费 |
| 适用 | 已有合作询盘的艺术家 |

## 3. 上架流程（验证后期）

```
艺术家档案完备（L1/L2）
        ↓
平台与艺术家共创：选 SKU 类型 + 定义交付物
        ↓
撰写预售页（叙事 + 权益边界 + 交付清单）
        ↓
人工审核（避免证券化/炒作表述）
        ↓
上线预售页（库存/名额/截止时间）
        ↓
收集支持 → 交付 → 记录进档案（L3/L6）
```

---

## 4. 预售页必备要素

- 链接到艺术家完整资产档案
- 关联系列/创作计划（L2 叙事）
- **交付物清单**（逐项列出）
- **不包含什么**（避免误解）
- 名额 / 截止时间
- 退款规则（如适用）
- 支持者将获得的可见记录（如编号、名单位）

---

## 5. 收费模型

| 项 | 说明 |
|----|------|
| 产品设计费 | 平台协助定义 SKU、撰写预售页 |
| 上架服务费 | 一次性或按次 |
| 成交抽成 | 支付接入后按成交额比例 |
| 艺术家会员 | 含 N 次预售配置额度（成长期） |

---

## 6. 风险与边界

| 风险 | 应对 |
|------|------|
| 证券化嫌疑 | 不卖「收益权」「份额」；优先权包需法务文案 |
| 交付纠纷 | 交付物清单书面化；支持记录可验证 |
| 炒作预期 | 禁止升值承诺；强调参与权与文献价值 |
| 艺术家品牌伤害 | 定制化审核；不强制所有艺术家都开预售 |

---

## 7. 验证期如何试点

1. 选 **1 位** 档案已完善的艺术家  
2. 选 **1 种** SKU（建议「系列文献包」）  
3. 人工制作预售页 + 线下/表单收款（可不接支付）  
4. 验证：是否有人愿意支持？艺术家是否认为有价值？交付是否顺畅？  
5. 再考虑产品化进系统（库存、订单、交付状态）

---

## 8. 与链上/NFT 的关系（后期可选）

若未来引入链上，仅作为：

- 文献包的编号凭证
- 支持关系的时间锚点
- 实体作品的映射证书

**不是**验证期目标，也**不是**产品化预售的必要条件。
